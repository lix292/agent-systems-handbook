## Summary

This starter extends the customer-support case study into a mailbox-integrated
local workflow: sync labeled Gmail threads, classify customer issues with
deterministic rules first, persist them in SQLite, and review or send replies
through a local dashboard.

## Status

`starter`

Source code: [case-studies/examples/customer-email-assist-starter](https://github.com/Prompthon-IO/agent-systems-handbook/tree/main/case-studies/examples/customer-email-assist-starter)

## Why It Exists

The earlier [Customer Support Email Agent Starter](/case-studies/examples/customer-support-email-agent-starter)
shows the safer draft-only boundary. This follow-on starter keeps the same
local-first posture, but adds the operational surfaces teams usually need next:
Gmail sync, local SQLite state, customer review queues, and a deterministic
dashboard.

## Quick Start

Install dependencies and run the local dashboard:

```bash
npm install
npm run setup-local
npm run dev
```

For deterministic sync commands:

```bash
npm run sync
tsx scripts/customer-email-assist.ts prepare-inbound-batch --out /tmp/prepared-inbound.json
tsx scripts/customer-email-assist.ts apply-send-queue
```

For the connector-assisted low-token path, write batches to files instead of
printing large JSON blobs to stdout:

```bash
tsx scripts/customer-email-assist.ts prepare-draft-batch --out /tmp/draft-batch.json
```

## Minimal-Token Boundary

This starter is designed to minimize model usage.

- Hard logic handles Gmail fetches, HTML stripping, signature trimming, quoted
  history removal, customer matching, SQLite persistence, analytics, and send
  queue execution.
- The intended skill workflow reserves model usage for only two JSON-only
  surfaces:
  - understanding cleaned inbound customer messages
  - generating template-field JSON for reply drafts
- The rendered reply text and HTML are produced locally from fixed templates,
  not from free-form model output.

## Included Sample Files

- `skill/SKILL.md`: Codex workflow for the two model-backed steps plus the
  deterministic CLI commands
- `src/lib/email-processing.ts`: HTML cleanup, quoted-history trimming, and
  heuristic classification helpers
- `src/lib/repository.ts`: SQLite queries for issues, customers, analytics, and
  review actions
- `src/lib/sync.ts`: deterministic send queue, inbound batch preparation, draft
  preparation, and draft rendering coordination
- `app/page.tsx`: Next.js App Router dashboard with issues, analysis, customer
  review, and customer CRUD tabs

## Constraints

- Gmail integration requires real OAuth environment variables.
- Attachments and inline images are ignored in v1.
- The dashboard is single-operator and local-machine oriented.
- `npm run sync` uses deterministic fallback understanding and drafting so the
  starter remains runnable without a live model call.
- `npm run setup-local` resets the local SQLite database to an empty state.
